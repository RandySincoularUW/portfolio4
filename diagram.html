System Architectural Diagram

This is an example of a software and hardware architectural diagram I created to show how ArcGIS Server and Portal for ArcGIS are deployed. These System Architectural diagrams are very important for educating support staff because they show server names, operating systems, software versions, etc.


Knowing what hardware and software components make up an application is very helpful when it comes time to patching servers with the latest security updates or determining what software versions are currently deployed. This documentation becomes even more critical when you have many applications to support and dozens of servers for each application.
